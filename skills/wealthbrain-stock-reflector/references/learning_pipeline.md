# 学习流水线

把复盘结果转化为更强选股能力时，使用本流程。

## 流水线

1. 原子经验：
   - 写一条短经验，并链接到决策和复盘。
   - 更新 `wiki/70-learning/lessons/lessons-log.md`。
   - 追加 `wiki/90-data/lessons.csv`。
2. 模式或错误：
   - 如果至少有三条相关经验，创建或更新 `wiki/70-learning/patterns/` 或 `wiki/70-learning/mistakes/`。
3. 待验证规则：
   - 把反复出现的模式转化为可证伪规则，写入 `wiki/70-learning/rules-under-test/`。
   - 追加或更新 `wiki/90-data/rules.csv`。
4. 方法论升级：
   - 只有证据支持的规则才能升级到 `wiki/20-methodology/playbooks/selection-playbook.md`。
   - 只有在有样本支持且记录了反例后，才把检查项加入 `wiki/20-methodology/checklists/master-investment-checklist.md`。

## 升级标准

- 1-2 个例子：只作为观察。
- 3 个以上相关例子：可以成为模式或错误。
- 5 个以上例子且考虑过反例：可以成为临时规则。
- 10 个以上例子，或有强一阶原理支撑：可以成为候选检查项。

## 反思问题

- 原始 thesis 是否可证伪？
- 检查清单是否抓住了主要风险？
- agent 是否把“故事好听”误当成“投资质量高”？
- 收益来自个股洞察，还是市场/行业 beta？
- 更优秀的投资者在决策前会检查什么？
- 未来 picker 的行为是否应该改变？
