# 知识库架构

WealthBrain 使用分层记忆，让推荐可以批量复盘，也让经验逐步升级为真正可用的投资检查清单。

## 分层

1. `10-policy/`：先定边界再找机会，定义投资范围、风险承受、仓位、流动性和系统禁止事项。
2. `20-methodology/`：可复用的决策逻辑，保存大师框架、检查清单、因子定义和选股打法。
3. `30-research/`：市场与行业背景，保存信息源记录、主题记录、行业记录和可复用研究素材。
4. `40-recommendations/`：决策日志，记录推荐了什么、何时推荐、参考价、理由和失效条件。
5. `50-stocks/`：个股记忆，每只股票一页，沉淀累计历史。
6. `60-reviews/`：结果记忆，保存单只股票和批量复盘。
7. `70-learning/`：学习管线，把复盘转化为经验、模式、错误和待验证规则。
8. `90-data/`：机器可读索引，CSV 文件是查找所有历史推荐的事实来源。

## 检索路径

### 推荐股票

使用 `$wealthbrain-stock-picker`：策略边界 -> 方法论检查清单 -> 既有经验 -> 个股页 -> 当前数据 -> 决策记录 -> CSV 索引。

### 查询全部历史推荐

使用 `$wealthbrain-stock-reflector`：`90-data/recommendations.csv` -> 当前价格 -> 批量复盘 -> 更新推荐、复盘、个股注册表和经验。

### 升级方法论

使用 `$wealthbrain-stock-reflector`：复盘 -> 经验 -> 模式/错误 -> 待验证规则 -> 升级检查清单项 -> 更新选股打法。

## 升级规则

- 观察：1-2 个案例，保存在 `70-learning/lessons/`。
- 候选规则：至少 3 个相关案例，保存在 `70-learning/rules-under-test/`。
- 临时检查项：至少 5 个案例，并考虑至少 1 个反例；加入 `20-methodology/checklists/`，状态标为 `provisional`。
- 核心检查项：至少 10 个跨市场环境案例，或有非常强的一阶原理支持；状态标为 `core`。

不要只因为某次盈利推荐“看起来很有说服力”就升级规则。
