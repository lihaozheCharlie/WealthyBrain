# RULE-20260708-001 机会成本校准

| 字段 | 说明 |
| --- | --- |
| 规则 ID | RULE-20260708-001 |
| 表述 | 当标的上行赔率明确、主要下行风险可用 `tiny/small` 仓位和失效条件控制时，不应因为证据未完全闭环而机械给 `watch`；应显式比较 `buy-candidate` 小仓位参与和完全等待的机会成本。 |
| 来源经验 | [[70-learning/lessons/lessons-log#LESSON-20260708-002]] |
| 样本数 | 0 个收益结果样本；当前仅为流程偏差观察 |
| 反例 | 暂无；后续需要记录“小仓位买入仍明显跑输”的案例 |
| 适用市场 | A股/港股/美股的中短期和中期研究建议 |
| 失效条件 | 若后续 `buy-candidate tiny/small` 明显增加亏损且没有提高相对收益，规则降级或废弃。 |
| 升级状态 | testing |

## 执行要求

每次给出 `watch` 时，必须写明：

- 不买的机会成本；
- 与现金、基准、同主题替代标的相比的预期收益差；
- 为什么不用 `tiny/small` 仓位表达参与；
- 触发升级为 `buy-candidate` 的最小证据。

## 复盘方式

在批量复盘中同时统计两类错误：

- 假阳性：给了 `buy-candidate` 后亏损或跑输基准；
- 假阴性：给了 `watch`，但标的随后明显上涨且原始风险没有发生。
